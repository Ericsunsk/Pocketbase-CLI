# PocketBase CLI v0.1.5

Release date: 2026-03-30

This release focuses on maintainability and installability rather than new runtime command surface.

## Highlights

- Development tooling is now aligned on current supported major versions for linting, bundling, and testing.
- The project now ships a one-line GitHub install/update script for easier first-run setup on new machines.
- README guidance now makes the installed command name and PATH expectations explicit.

## Changes Included

- Upgraded `eslint` to v9 and migrated the repository to flat-config `eslint.config.mjs`.
- Upgraded `@typescript-eslint` to v8.
- Upgraded `tsup` to v8.5.1.
- Upgraded `vitest` to v4.1.2.
- Added `scripts/install-global.sh` to clone or update the repository, build it, install `pocketbase-cli`, and print a PATH hint when needed.
- Refined test linting to preserve useful signal without forcing repetitive return type annotations on routine test context builders.

## Validation

The release candidate was validated with:

- `npm install`
- `npm run typecheck`
- `npm run lint`
- `npm test`
- `npm run build`

## Suggested Release Summary

PocketBase CLI `v0.1.5` is a maintenance release that modernizes the development toolchain, migrates linting to ESLint flat config, and adds a one-line GitHub install script so new machines can bootstrap the CLI faster.
