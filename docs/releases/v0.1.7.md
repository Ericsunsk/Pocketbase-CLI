# PocketBase CLI v0.1.7

Release date: 2026-04-02

This release streamlines the authentication surface by converging browser login behavior under a single `auth login` command and synchronizes the docs with the updated workflow.

## Highlights

- Unified browser auth command: use `auth login` for the local loopback browser flow.
- Removed the separate `auth login-browser` command path to reduce command-surface ambiguity.
- Updated install/uninstall documentation and release references across multilingual README guides.

## Changes Included

### Auth command flow

- Renamed browser-login command registration from `auth.login-browser` to `auth.login`.
- Updated auth login examples and action/history metadata to the unified command name.
- Kept `--no-open` and timeout controls for headless and CI-friendly environments.

### Documentation

- Refreshed `README.md`, `README.en.md`, and `README.zh-CN.md` to match the current auth UX and simplify uninstall instructions.
- Updated release/supporting docs touched in this cycle (`CHANGELOG.md`, `DEVELOPMENT.md`, `FEATURES.md`, `TESTING.md`, `docs/releases/v0.1.3.md`).

## Validation

The release candidate was validated with:

- `npm run lint`
- `npm run typecheck`
- `npm test`
- `npm run build`

## Suggested Release Summary

PocketBase CLI `v0.1.7` unifies browser authentication under `auth login`, removes the redundant `auth login-browser` path, and aligns multilingual documentation with the new workflow for a cleaner operator and automation experience.
