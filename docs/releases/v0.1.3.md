# PocketBase CLI v0.1.3

Release date: 2026-03-28

This release hardens authentication and local state handling, tightens remote target validation, and refreshes the project documentation for a formal release workflow.

## Highlights

- Encrypted persisted local session storage at rest, with a sibling `session.json.key` file used for local decryption.
- Stricter base URL validation across config parsing, preflight checks, info reporting, and shared remote client setup.
- Clearer authentication guidance centered on command arguments, `--password-stdin`, and `auth login-browser`.
- Refined browser-assisted login behavior for both interactive and headless environments.
- Release-style documentation refresh across product, development, testing, and changelog files.

## Behavioral Changes

- `POCKETBASE_CLI_BASE_URL` and `POCKETBASE_CLI_STATE_DIR` remain supported for environment-based defaults.
- Environment-based credential fallbacks are intentionally unsupported. Use command arguments, `--password-stdin`, or `auth login-browser`.
- Malformed remote base URLs are rejected earlier and more consistently, including values with embedded credentials, query strings, or fragments.
- Stored auth state is cleared when persisted target settings no longer match the current remote target.

## Compatibility Notes

- Existing plaintext session files continue to load.
- Newly written session state is stored in encrypted form.
- Browser-assisted login remains local-only and uses a temporary loopback server on `127.0.0.1`.

## Validation

The release candidate was validated with:

- `npm run typecheck`
- `npm test`
- `npm run build`

## Suggested Release Summary

PocketBase CLI `v0.1.3` focuses on safer authentication workflows and stronger local state protection. It introduces encrypted persisted session storage, consistent base URL validation across the CLI, clarified credential handling, and a release-grade documentation refresh. This version is a recommended upgrade for users relying on remote administration, automation, and agent-driven workflows against deployed PocketBase instances.
