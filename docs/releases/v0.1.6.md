# PocketBase CLI v0.1.6

Release date: 2026-03-31

This release improves the machine-readable `schema --json` contract so that LLM agents can use the CLI with fewer round-trips and less guesswork.

## Highlights

- Every command now ships with at least one usage example in the schema contract.
- PocketBase filter and sort syntax is documented inline, eliminating the need for external reference lookups.
- Multi-step authentication flows (MFA, OTP, password reset, verification, email change) include notes describing the full workflow sequence.
- The redundant `dangerous` field has been removed from the schema contract — use `destructive` instead.

## Changes Included

### Schema contract

- Removed the `dangerous` field from `SchemaEntry` (was an exact copy of `destructive`).
- Marked `backups.download --token` as `sensitive: true`.

### Command metadata

- Added usage examples to `backups list`, `backups create`, `backups upload`, `backups delete`, `backups download`, `backups restore`, `records delete`, `records delete-by-filter`, `records auth-otp`, `records request-otp`, `records auth-methods`, `records request-password-reset`, `records confirm-password-reset`, `records request-verification`, `records confirm-verification`, `records request-email-change`, `records confirm-email-change`, `collections delete`, `collections truncate`, and `auth logout`.
- Added `help` text to all inline parameter definitions in destructive commands that previously lacked it.
- Added `notes` documenting multi-step flows: MFA continuation via `--mfa-id`, OTP two-step flow (`request-otp` → `auth-otp`), and request-then-confirm patterns for password reset, verification, and email change.
- Added PocketBase filter/sort syntax documentation to `records.list` notes and improved `--filter`/`--sort` help text across records and collections commands.

## Validation

The release candidate was validated with:

- `npm run typecheck`
- `npm test` (115 tests passed)
- `npm run build`

## Suggested Release Summary

PocketBase CLI `v0.1.6` enriches the `schema --json` contract with usage examples on every command, inline filter/sort syntax documentation, multi-step auth flow notes, and cleaner metadata — making the CLI significantly easier for LLM agents to use with fewer token-consuming round-trips.
